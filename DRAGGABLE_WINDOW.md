## 🔧 Технические детали

### Что было добавлено в код:

```cpp
case WM_NCHITTEST:
    {
        // Allow dragging window by clicking anywhere
        LRESULT hit = DefWindowProc(hWnd, msg, wParam, lParam);
        if (hit == HTCLIENT)
            return HTCAPTION; // Treat client area as title bar for dragging
        return hit;
    }
```

### Как это работает:

1. **WM_NCHITTEST** - сообщение Windows для определения области окна
2. **HTCLIENT** - клиентская область (содержимое окна)
3. **HTCAPTION** - заголовок окна (область для перетаскивания)

**Трюк:** Мы "обманываем" Windows, говоря что вся клиентская область - это заголовок окна. Поэтому Windows автоматически обрабатывает перетаскивание!

## ⚙️ Совместимость с ImGui

### Не мешает работе ImGui:
- ✅ Кнопки кликаются
- ✅ Читы переключаются
- ✅ ESC закрывает окно
- ✅ Горячие клавиши работают

### Почему не конфликтует:
ImGui обрабатывает события первым:
```cpp
if (ImGui_ImplWin32_WndProcHandler(hWnd, msg, wParam, lParam))
    return true; // ImGui handled it
```

Только если ImGui **не** обработал событие, мы обрабатываем `WM_NCHITTEST`.

## 🐛 Возможные проблемы

### Проблема: Не перетаскивается
**Причина:** Старая версия .exe  
**Решение:** Пересоберите проект:
```powershell
.\build_signed_only.ps1
```

### Проблема: Окно "прыгает" при клике
**Причина:** Конфликт с ImGui элементами  
**Решение:** Это нормально - ImGui обрабатывает клики на своих элементах

### Проблема: Окно перетаскивается случайно
**Решение:** Это особенность дизайна - вся область окна draggable
