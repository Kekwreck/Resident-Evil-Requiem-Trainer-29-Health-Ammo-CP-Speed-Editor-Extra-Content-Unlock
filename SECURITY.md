# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.6   | :white_check_mark: |
| 1.0.5   | :white_check_mark: |
| < 1.0.5 | :x:                |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

### How to Report

1. **Email:** Send details to [security@example.com]
2. **Expected Response:** Within 48 hours
3. **Updates:** Every 7 days until resolved

### What to Include

- Type of vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Process

1. **Report received** - Acknowledged within 48h
2. **Investigation** - We assess the issue
3. **Fix developed** - Patch created
4. **Release** - Security update published
5. **Disclosure** - Public announcement

## Security Measures

### Current

- ✅ No network connections
- ✅ No data collection
- ✅ Runs as normal user
- ✅ Open source code
- ✅ Digital signature

### Planned

- 🔄 Code signing with trusted CA
- 🔄 Automated security scanning
- 🔄 Regular security audits

## Known Limitations

- Self-signed certificate (not trusted by Windows)
- Antivirus may flag (false positive)
- Requires game process access

## Best Practices

### For Users

- Download only from official GitHub releases
- Verify file checksums
- Use signed version for production
- Keep trainer updated

### For Developers

- Review all code changes
- Test security implications
- Follow secure coding guidelines
- Report issues privately
